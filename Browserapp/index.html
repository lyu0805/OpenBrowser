新建自定义模板

                
把可重复执行的浏览器操作保存为本机模板，后续可随时创建流程、修改、导入或导出。

              

              

                
              

            


            

              

                新建模板
                步骤 JSON
                保存和使用
                运行流程
                导入和导出
              

              

                

                  
1. 新建模板

                  
在模板商店点击「定制模板」。填写模板名称、分类和说明后，在“步骤 JSON”中填写需要执行的操作。

                  
    
                    
  • 模板名称：用于在商店中识别模板，例如“每日签到”。
    • 
                    
  • 分类：用于筛选，建议按业务或网站命名，例如“内容发布”。
    • 
                    
  • 说明：写明用途、前置条件或需要登录的账号状态。
    • 
                  

                


                

                  
2. 编写步骤 JSON

                  
步骤 JSON 必须是一个数组，数组中的每项代表一条按顺序执行的操作。以下示例依次打开网页、等待、点击链接并输入内容：

                  
[
  { "type": "goto", "url": "https://example.com" },
  { "type": "wait", "ms": 1000 },
  { "type": "click", "selector": "a" },
  { "type": "type", "selector": "input", "text": "hello", "human": true }
]

                  

                    
goto打开页面，使用 url 指定网址。

                    
wait等待页面或元素加载，使用 ms 指定毫秒数。

                    
click点击元素，使用 CSS 选择器 selector 定位元素。

                    
type输入文字，使用 selector 定位输入框，使用 text 指定内容。

                  

                  
human: true 会逐字输入文字。选择器应先在测试环境验证，页面加载较慢时应在操作前加入 wait。

                


                

                  
3. 保存和使用

                  
点击「保存到商店」后，模板会出现在模板商店的来源「我的/导入」中。点击模板卡片上的「使用」会创建一份新的可编辑流程，原模板不会被修改。

                  
可以在“流程管理”中编辑新流程的名称、运行环境和步骤 JSON；确认无误后再保存或运行。

                


                

                  
4. 运行流程

                  
进入「流程管理」，选择要运行的浏览器环境。环境需要先在“环境管理”中启动。选择环境后，点击「运行」即可按顺序执行步骤。

                  
建议先在测试环境验证网页结构、选择器和等待时间。运行结果和错误信息可在运行日志与运行记录中查看。

                


                

                  
导入、导出和本机次数

                  
「导出我的模板」会生成 JSON 备份文件；「导入 JSON」可恢复备份或导入其他设备创建的自定义模板。模板与使用数据均保存在当前电脑。

                  
商店显示的“本机使用次数”只会在本机点击「使用」并创建流程时增加，不代表公开使用量或其他人的使用数据。
